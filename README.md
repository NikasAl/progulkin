# Прогулкин 🚶

**Прогулкин** — приложение для трекинга прогулок, которое превращает обычную ходьбу в увлекательный процесс исследования и улучшения окружающего мира.

## Концепция

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   "Превращаем прогулки в приключения,                       │
│    а уборку мусора — в игру"                                │
│                                                             │
│   ┌─────────┐     ┌─────────┐     ┌─────────┐               │
│   │ Гуляем  │ ──→ │ Находим │ ──→ │ Улучшаем│               │
│   │         │     │ объекты │     │  мир    │               │
│   └─────────┘     └─────────┘     └─────────┘               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

Пользователи могут создавать и находить различные объекты на карте во время прогулок:
- **👹 Мусорные монстры** — отмечаем мусор, убираем, получаем очки
- **📜 Секретные сообщения** — послания, которые читаются только в определённом месте
- **🦊 Существа** — коллекционные цифровые существа из русской мифологии
- **📝 Заметки о местах** — делимся интересными находками с фото
- **🔔 Напоминалки** — гео-напоминания с персонажами Смешариков

## Децентрализация данных

### Философия приватности

**Прогулкин** не хранит данные пользователей на серверах. Все данные находятся на устройствах пользователей и синхронизируются напрямую между ними.

```
┌─────────────────────────────────────────────────────────────┐
│                    Традиционные приложения                  │
│                                                             │
│   Пользователь ──→ Сервер компании ──→ Пользователь         │
│                        ▲                                    │
│                        │                                    │
│              Все данные здесь                               │
│              (компания видит всё)                           │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                      Прогулкин (P2P)                        │
│                                                             │
│   Пользователь A ←──P2P──→ Пользователь B                   │
│         ↑                          ↑                        │
│         │                          │                        │
│         └──────── Signaling ───────┘                        │
│                  (только знакомит)                          │
│                                                             │
│   Данные хранятся только на устройствах пользователей       │
└─────────────────────────────────────────────────────────────┘
```

### Что хранится и где

| Где | Что хранится |
|-----|--------------|
| **Устройство пользователя** | Маршруты, объекты, настройки, статистика, фото |
| **Signaling сервер** | Временный ID устройства, зона (geohash), IP:порт |
| **Нигде** | Имена, email, пароли, личные данные |

### Преимущества подхода

- ✅ **Полная приватность** — ваши данные только у вас
- ✅ **Нет регистрации** — не нужен email или пароль
- ✅ **Работает офлайн** — все функции доступны без интернета
- ✅ **Нет цензуры** — децентрализованная сеть не контролируется
- ✅ **Долгосрочность** — данные не исчезнут при закрытии сервиса

## P2P Синхронизация

### Как это работает

```
1. Пользователь открывает приложение
   └── Подключается к signaling серверу
   └── Регистрируется в своей географической зоне

2. Signaling сервер знакомит устройства
   └── Отправляет список IP:порт устройств в той же зоне
   └── Не хранит и не видит данные объектов

3. Устройства соединяются напрямую (P2P)
   └── Обмениваются объектами карты
   └── Синхронизируют новые данные

4. При потере связи
   └── Данные сохраняются локально
   └── Синхронизация возобновляется при восстановлении связи
```

### Географические зоны

Для эффективной синхронизации мир делится на зоны по Geohash:

```
Точность geohash:
├── 4 символа: ~20 км (город)
├── 5 символов: ~5 км (район)
├── 6 символов: ~1.2 км (квартал) ← используем
└── 7 символов: ~150 м (дом)

Синхронизация происходит только с устройствами в той же зоне
или соседних зонах (радиус ~10 км).
```

### Signaling сервер

Минимальный сервер, который только «знакомит» устройства:

```bash
# Запуск на своём сервере
cd progulkin
dart run bin/signaling_server.dart --port 9000
```

**Что делает сервер:**
- Регистрирует устройства в географических зонах
- Сообщает IP:порт устройств для P2P соединения
- Помогает с UDP Hole Punching для обхода NAT

**Что НЕ делает сервер:**
- ❌ Не хранит объекты карты
- ❌ Не хранит маршруты
- ❌ Не хранит личные данные
- ❌ Не модерирует контент

## Объекты на карте

### 👹 Мусорные монстры

Превращаем проблему мусора в игру. Каждый мусорный объект — это «монстр», которого нужно «победить» (убрать).

| Класс | Сложность | Примеры | Очки |
|-------|-----------|---------|------|
| 🟢 1 | Взять и нести | Бумажка, бутылка | 10 |
| 🟡 2 | Нужен пакет | 3-5 бутылок | 20 |
| 🟠 3 | Нужны инструменты | Шины, техника | 30 |
| 🔴 4 | Нужна помощь | Мебель | 40 |
| 💀 5 | Нужна организация | Свалка | 50 |

**Типы мусора:** 🍺 Бутылки, 🗞️ Бумага, 🛞 Шины, 🛋️ Мебель, 🏗️ Строймусор, 📺 Электроника, 🥤 Пластик, 🍂 Органика

### 📜 Секретные сообщения

Сообщения, которые можно прочитать только находясь в определённом месте.

**Типы:** 📜 Текст, ❓ Загадка, 📖 История, 💫 Пожелание, 💡 Совет, 💭 Воспоминание

**Механика:**
1. Создаёте секрет в определённом месте
2. Указываете радиус разблокировки (10-100м)
3. Другие видят только заголовок
4. Чтобы прочитать — нужно прийти на место

### 🦊 Существа русской мифологии

Коллекционные цифровые существа, появляющиеся в определённых местах.

| Редкость | Существа |
|----------|----------|
| ⚪ Обычные | 🏠 Домовой, 🐠 Золотая рыбка |
| 🟢 Необычные | 🌿 Кикимора, 🧜‍♀️ Русалка |
| 🔵 Редкие | 🌲 Леший, 🌊 Водяной |
| 🟣 Эпические | 🧙‍♀️ Баба Яга, 🦅 Сирин, 🐦 Алконост |
| 🟡 Легендарные | 🐉 Змей Горыныч, 🔥 Жар-птица |
| 🔴 Мифические | 💀 Кощей |

**Механика:** Существа появляются в подходящих местах (лес, вода, город) на 30-120 минут. Можно «поймать» и собрать коллекцию.

### 📝 Заметки о местах

Делитесь интересными находками с другими пользователями.

**Категории:**
- 🐿️ **Природа** — животные, растения, природные явления
- 📚 **Культура** — исторические места, достопримечательности
- 🏃 **Спорт** — спортивные площадки, маршруты
- ☕ **Еда** — кафе, магазины, рынки
- 📸 **Точки съёмки** — лучшие ракурсы для фото
- 🎨 **Творчество** — стрит-арт, музеи под открытым небом
- 🎮 **Игры** — игровые площадки, квесты
- 💡 **Советы** — полезные подсказки
- ❓ **Другое** — всё остальное

**Особенности:**
- Фото с камеры (только на месте заметки, GPS-верификация)
- WebP сжатие (150-250KB для оригинала, 30-50KB для превью)
- Кнопка "Интересно" для связи по общим интересам

### 🔔 Напоминалки

Гео-напоминания с персонажами Смешариков.

**Персонажи:** 🐻 Копатыч, 🐰 Крош, 🦔 Ёжик, 🐷 Нюша, 🦉 Кар-Карыч, 🦌 Лосяш, 🐧 Пин, 🐑 Бараш

**Механика:**
1. Создаёте напоминание в определённом месте
2. Выбираете персонажа
3. Указываете радиус срабатывания
4. При приближении получаете уведомление

**Функции:**
- ✅ Включение/отключение напоминаний
- ✅ Откладывание (15 мин, 30 мин, 1ч, 2ч, до завтра)
- ✅ Счётчик срабатываний
- ✅ Статус бейджи на карте (активно/отложено/отключено)
- ✅ Push-уведомления через flutter_local_notifications

## Установка

### Требования

- Flutter SDK 3.0+
- Android SDK 21+ (Android 5.0+)


## Безопасность контента

В P2P сети модерация невозможна центральным образом, поэтому:

- ✅ **Фото только с камеры** — GPS верификация места съёмки
- ✅ **WebP сжатие** — минимальный размер, нет EXIF данных
- ✅ **Локальный фильтр** — запрещённые слова блокируются
- ✅ **Репутация** — объекты новичков скрыты по умолчанию
- ✅ **Голосование** — пользователи подтверждают или опровергают объекты
- ✅ **Автоскрытие** — объекты с жалобами скрываются

## Политика конфиденциальности

Для магазинов приложений (RuStore, Google Play):

```
1. Все данные хранятся на устройстве пользователя
2. P2P синхронизация — пользователь контролирует
3. Нет регистрации и личных данных
4. Signaling сервер только знакомит устройства
5. Можно отключить синхронизацию в настройках
```
