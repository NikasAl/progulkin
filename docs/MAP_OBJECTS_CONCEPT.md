# Прогулкин: Система объектов карты
## Концепция и дорожная карта реализации

---

## 1. Общая концепция

### 1.1. Идея

**Прогулкин** — приложение для трекинга прогулок, которое превращает обычную ходьбу в увлекательный процесс исследования и улучшения окружающего мира.

**Ключевая идея:** Пользователи могут создавать и находить различные объекты на карте во время прогулок. Объекты синхронизируются между пользователями через децентрализованную P2P сеть без центрального сервера хранения данных.

### 1.2. Философия

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   "Превращаем прогулки в приключения,                       │
│    а уборку мусора — в игру"                                │
│                                                             │
│   ┌─────────┐     ┌─────────┐     ┌─────────┐              │
│   │ Гуляем  │ ──→ │ Находим │ ──→ │ Улучшаем│              │
│   │         │     │ объекты │     │  мир    │              │
│   └─────────┘     └─────────┘     └─────────┘              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.3. Принципы

| Принцип | Описание |
|---------|----------|
| **Приватность** | Все данные хранятся на устройстве пользователя |
| **Децентрализация** | P2P синхронизация без центрального хранилища |
| **Безопасность** | Нет фото — нет неприемлемого контента |
| **Расширяемость** | Легко добавлять новые типы объектов |
| **Игровизация** | Очки, уровни, достижения |

## 2. Типы объектов

### 2.1. Мусорные монстры (TrashMonster)

**Концепция:** Превращаем проблему мусора в игровую задачу. Каждый мусорный объект на карте — это «монстр», которого нужно «победить» (убрать).

**Мотивация:** Люди охотнее убирают мусор, если это часть игры с очками и достижениями.

#### Классификация монстров

| Класс | Название | Сложность уборки | Примеры | Очки |
|-------|----------|------------------|---------|------|
| 1 | Лёгкий 🟢 | Взять и нести | Бумажка, бутылка | 10 |
| 2 | Средний 🟡 | Нужен пакет | Куча бумаги, 3-5 бутылок | 20 |
| 3 | Сложный 🟠 | Нужны инструменты | Шины, мелкая техника | 30 |
| 4 | Тяжёлый 🔴 | Нужна помощь/транспорт | Мебель, крупная техника | 40 |
| 5 | Босс 💀 | Нужна организация | Стихийная свалка | 50 |

#### Типы мусора

```
🍺 Бутылки/банки     — самый частый тип
🗞️ Бумага/картон     — упаковка, газеты
🛞 Шины              — часто в лесополосах
🛋️ Мебель            — диваны, стулья
🏗️ Строймусор        — остатки ремонта
📺 Электроника       — старая техника
🥤 Пластик           — упаковка, контейнеры
🍂 Органика          — пищевые отходы
🗑️ Смешанный         — непонятный мусор
❓ Другое            — неопределимое
```

#### Механика

```
1. Пользователь находит мусор
2. Отмечает на карте: тип, количество, описание
3. Система определяет класс монстра
4. Другие пользователи видят монстра на карте
5. Кто-то приходит и убирает → отмечает как убранный
6. Получает очки × класс × количество
```

### 2.2. Секретные сообщения (SecretMessage)

**Концепция:** Сообщения, которые можно прочитать только находясь в определённом месте. Цифровой аналог «тайников» и геокешинга.

**Мотивация:** Стимулирует исследовать новые места, создавать и находить скрытые послания.

#### Типы секретов

| Тип | Описание | Пример |
|-----|----------|--------|
| 📜 Текст | Простое сообщение | «Здесь я сделал предложение» |
| ❓ Загадка | Загадка с ответом | «Найди дерево с дуплом...» |
| 📖 История | История места | «В этом доме жил...» |
| 💫 Пожелание | Пожелание месту | «Пусть здесь всегда светит солнце» |
| 💡 Совет | Полезный совет | «Лучшая точка для заката» |
| 💭 Воспоминание | Личное воспоминание | «Здесь мы познакомились» |

#### Механика

```
1. Пользователь создаёт секрет в определённом месте
2. Устанавливает радиус разблокировки (10-100м)
3. Выбирает тип и пишет текст
4. Другие пользователи видят только заголовок и иконку
5. Чтобы прочитать — нужно прийти на место
6. Опционально: одноразовые (исчезают после прочтения)
```

#### Варианты использования

```
Личные:
- «Памятные места отношений»
- «Любимые точки для фото»
- «Маршрут первого свидания»

Развлекательные:
- Городские квесты
- Тайники с подсказками
- Загадки для друзей

Полезные:
- «Здесь удобная лавочка»
- «Отличный вид на закат»
- «Скрытый проход во двор»
```

### 2.3. Существа (Creature)

**Концепция:** Коллекционные цифровые существа из русской мифологии. Появляются в определённых местах на ограниченное время. Можно «поймать» и собрать коллекцию.

**Мотивация:** Стимулирует регулярные прогулки и исследование разных мест (по аналогии с Pokémon GO, но с русским колоритом).

#### Иерархия существ

```
Обычные (white) ⚪
├── 🏠 Домовой — дома, улицы с жилыми домами
└── 🐠 Золотая рыбка — водоёмы

Необычные (green) 🟢
├── 🌿 Кикимора — болота, влажные места
└── 🧜‍♀️ Русалка — реки, озёра

Редкие (blue) 🔵
├── 🌲 Леший — леса, парки
└── 🌊 Водяной — глубокие водоёмы

Эпические (purple) 🟣
├── 🧙‍♀️ Баба Яга — леса, окраины
├── 🦅 Сирин — поля, возвышенности
└── 🐦 Алконост — берега рек

Легендарные (yellow) 🟡
├── 🐉 Змей Горыныч — горы, холмы
└── 🔥 Жар-птица — поляны, опушки

Мифические (red) 🔴
└── 💀 Кощей — руины, заброшенные места
```

#### Среда обитания

```
🌲 Лес      — Леший, Кикимора, Баба Яга
💧 Вода     — Водяной, Русалка, Золотая рыбка
🌿 Болото   — Кикимора, Водяной
⛰️ Горы     — Змей Горыныч
🌾 Поле     — Сирин, Жар-птица
🏙️ Город    — Домовой
🏠 Дом      — Домовой
🌍 Везде    — Кощей (редко)
```

#### РПГ-элементы

```
Характеристики:
├── Здоровье (HP)
├── Атака
├── Защита
└── Способности (1-3)

Пример способностей:
├── Домовой: «Прятки», «Тепло дома», «Защита»
├── Леший: «Лесное зрение», «Морока», «Прохлада»
├── Змей Горыныч: «Огненное дыхание», «Три головы», «Полёт»
└── Жар-птица: «Пламя», «Сияние», «Перо удачи»
```

#### Механика появления

```
Время жизни: 30-120 минут (зависит от редкости)
Спавн: зависит от среды обитания + время суток
├── Домовой: чаще вечером
├── Леший: в лесу, любое время
├── Водяной: у воды, ночью активнее
└── Жар-птица: на рассвете/закате

Появление:
1. Система генерирует существо в подходящем месте
2. Пользователи в зоне получают уведомление
3. Кто первый пришёл — может «поймать»
4. После поимки добавляется в коллекцию
```

## 3. P2P Синхронизация

### 3.1. Архитектура

```
┌─────────────────────────────────────────────────────────────┐
│                     P2P Network Topology                    │
│                                                             │
│                      Signaling Server                       │
│                    (только знакомит устройства)             │
│                            ▲                                │
│                            │ TCP                            │
│                            │                                │
│         ┌──────────────────┼──────────────────┐            │
│         │                  │                  │            │
│         ▼                  ▼                  ▼            │
│    ┌─────────┐        ┌─────────┐        ┌─────────┐      │
│    │ Device A│◄──────►│ Device B│◄──────►│ Device C│      │
│    │  Zone 1 │  UDP   │  Zone 1 │  UDP   │  Zone 1 │      │
│    └─────────┘        └─────────┘        └─────────┘      │
│         │                                            │      │
│         │              Zone 2                        │      │
│         │    ┌─────────┐        ┌─────────┐         │      │
│         └───►│ Device D│◄──────►│ Device E│◄────────┘      │
│              └─────────┘  UDP   └─────────┘                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2. Роли сервера

**Signaling Server (сигнальный сервер):**

| Делает | НЕ делает |
|--------|-----------|
| Регистрирует устройства в зонах | Хранит объекты |
| Сообщает IP:port устройств | Хранит маршруты |
| Помогает с NAT traversal | Хранит личные данные |
| Поддерживает список активных пиров | Модерирует контент |

**Почему это безопасно для приватности:**
- Сервер знает только зону (geohash 6 символов ≈ 1.2км)
- Сервер знает только временный ID устройства
- Сервер НЕ знает координаты объектов
- Сервер НЕ видит содержимое объектов

### 3.3. Geohash зоны

```
Точность geohash:
├── 4 символа: ~20 км (город)
├── 5 символов: ~5 км (район)
├── 6 символов: ~1.2 км (квартал) ← используем
└── 7 символов: ~150 м (дом)

Пример для Москвы:
├── u4pry1 — Красная площадь
├── u4pryz — Александровский сад
└── u4pryw — Зарядье
```

### 3.4. Протокол синхронизации

```
1. Device A входит в зону
   └── Подключается к signaling
   └── Регистрируется: {zone: "u4pry", port: 9001}

2. Signaling отправляет список пиров
   └── {type: "peers", peers: [{id: "B", ip: "x.x.x.x", port: 9002}]}

3. Device A соединяется с Device B (UDP)
   └── Отправляет: {type: "sync_request", objects: [...]}
   └── Получает: {type: "sync_data", objects: [...]}

4. Оба мержат полученные объекты
   └── Правило: версия выше — побеждает
   └── При конфликте timestamp — последний побеждает
```

### 3.5. Offline-режим

```
Приложение работает полностью офлайн:

1. Все объекты в локальной SQLite базе
2. Маршруты сохраняются локально
3. При появлении сети — синхронизация

Синхронизация при восстановлении связи:
1. Запросить объекты в своей зоне
2. Отправить свои несинхронизированные объекты
3. Мерж с разрешением конфликтов
```

## 6. Технические детали

### 6.1. Размер данных

```
TrashMonster:    ~300-500 байт
SecretMessage:   ~500-1000 байт
Creature:        ~400-600 байт

При 1000 объектов: ~500 KB — ~1 MB
При 10000 объектов: ~5-10 MB
```

### 6.2. Сетевой трафик

```
Синхронизация с 1 пиром:
├── Запрос: ~10-50 KB
├── Ответ: ~10-50 KB
└── Итого: ~20-100 KB

При 10 пирах одновременно: ~1 MB
Периодичность: каждые 30 секунд
Трафик в час: ~120 MB (активный режим)
```

### 6.3. Батарея

```
Оптимизации:
├── Синхронизация только при активном приложении
├── Увеличение интервала при низком заряде
├── Пауза при отсутствии движения
└── Фоновый режим только для геолокации
```

### 6.4. Безопасность

```
Данные:
├── Не передаются пароли
├── Не передаются личные данные
├── Нет фото — нет неприемлемого контента
└── Текст фильтруется локально

P2P:
├── Шифрование TLS (опционально)
├── Проверка подписи сообщений (опционально)
└── Игнорирование забаненных пользователей

Репутация:
├── Объекты новичков скрыты по умолчанию
├── Система голосов: подтверждают/отвергают
├── Автоматическое скрытие при жалобах
└── Бан при систематических нарушениях
```

## 7. Конфиденциальность

### 7.1. Что хранится

| Где | Что |
|-----|-----|
| Устройство | Маршруты, объекты, настройки |
| Signaling server | Временный ID, зона (geohash), IP:порт |
| Нигде | Имена, email, пароли, личные данные |

### 7.2. Что передаётся

| Кому | Что |
|------|-----|
| P2P другим | Объекты (координаты, тип, описание) |
| Signaling | Зона, порт |
| OSM | Запросы тайлов карты |

### 7.3. Политика для магазинов

```
Для Google Play и RuStore:

1. Все данные на устройстве пользователя
2. P2P синхронизация — пользователь контролирует
3. Нет регистрации и личных данных
4. Сервер только знакомит устройства
5. Можно отключить синхронизацию
```

## 9. Риски и митигация

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Нет пользователей рядом | Высокая | Среднее | Offline режим, bootstrap ноды |
| Спам объектами | Средняя | Среднее | Репутация, подтверждения |
| Расход батареи | Средняя | Высокое | Оптимизация интервалов |
| NAT не пробивается | Низкая | Высокое | STUN серверы, relay |
| Конфликты данных | Низкая | Низкое | CRDT, version vectors |

## 10. Заключение

Система объектов карты превращает **Прогулкин** из простого трекера прогулок в **платформу для исследования и улучшения окружающего мира**.

Ключевые преимущества:

1. **Приватность** — нет центрального хранилища данных
2. **Безопасность** — нет фото, текст фильтруется
3. **Расширяемость** — легко добавлять новые типы объектов
4. **Офлайн** — работает без интернета
5. **Социальность** — синхронизация с соседними пользователями
6. **Игровизация** — очки, уровни, достижения

Начинаем с мусорных монстров как самого полезного типа объектов, затем добавляем секреты и существа для увлекательности.
