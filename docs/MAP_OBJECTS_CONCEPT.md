# Прогулкин: Система объектов карты
## Концепция и дорожная карта реализации

---

## 1. Общая концепция

### 1.1. Идея

**Прогулкин** — приложение для трекинга прогулок, которое превращает обычную ходьбу в увлекательный процесс исследования окружающего мира и поиска единомышленников.

**Ключевая идея:** Пользователи могут создавать и находить различные объекты на карте во время прогулок. Объекты синхронизируются между пользователями через децентрализованную P2P сеть без центрального сервера хранения данных.

### 1.2. Философия (обновлено)

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   "Превращаем прогулки в приключения,                       │
│    находим единомышленников по интересам"                   │
│                                                             │
│   ┌─────────┐     ┌─────────┐     ┌─────────┐              │
│   │ Гуляем  │ ──→ │Отмечаем │ ──→ │Находим  │              │
│   │         │     │интересное│    │ друзей  │              │
│   └─────────┘     └─────────┘     └─────────┘              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.3. Принципы

| Принцип | Описание |
|---------|----------|
| **Приватность** | Все данные хранятся на устройстве пользователя |
| **Децентрализация** | P2P синхронизация без центрального хранилища |
| **Безопасность** | Фото с модерацией, текст фильтруется |
| **Интересы** | Социальные связи формируются вокруг общих интересов |
| **Расширяемость** | Легко добавлять новые типы объектов |
| **Игровизация** | Очки, уровни, достижения |

## 2. Типы объектов

### 2.1. Мусорные монстры (TrashMonster) ✅ РЕАЛИЗОВАНО

**Концепция:** Превращаем проблему мусора в игровую задачу. Каждый мусорный объект на карте — это «монстр», которого нужно «победить» (убрать).

#### Классификация монстров

| Класс | Название | Сложность уборки | Примеры | Очки |
|-------|----------|------------------|---------|------|
| 1 | Лёгкий 🟢 | Взять и нести | Бумажка, бутылка | 10 |
| 2 | Средний 🟡 | Нужен пакет | Куча бумаги, 3-5 бутылок | 20 |
| 3 | Сложный 🟠 | Нужны инструменты | Шины, мелкая техника | 30 |
| 4 | Тяжёлый 🔴 | Нужна помощь/транспорт | Мебель, крупная техника | 40 |
| 5 | Босс 💀 | Нужна организация | Стихийная свалка | 50 |

#### Уникальные изображения для каждого типа

Для каждого типа мусора генерируются уникальные изображения монстров:

| Тип мусора | Визуальный стиль монстра |
|------------|-------------------------|
| 🍺 Бутылки | Стеклянный монстр с блестящими осколками |
| 🗞️ Бумага | Бумажный голем из скомканных листов |
| 🛞 Шины | Резиновый великан с колесами-руками |
| 🛋️ Мебель | Древесный голем из старой мебели |
| 🏗️ Строймусор | Бетонный монстр с арматурой |
| 📺 Электроника | Механический робот из плат и проводов |
| 🥤 Пластик | Пластиковый дракон из бутылок |
| 🍂 Органика | Грязевое чудовище |
| 🗑️ Смешанный | Химера из разных материалов |
| ❓ Другое | Таинственный туманный монстр |

### 2.2. Заметки об интересных местах (InterestNote) 🆕

**Концепция:** Пользователи отмечают интересные места и находят единомышленников через общие интересы. Не "гулять с кем попало", а находить людей с похожими интересами.

**Ключевое отличие:** Связь формируется через интерес к месту/явлению, а не просто географическую близость.

#### Примеры использования

| Категория | Пример заметки | Как формируется связь |
|-----------|----------------|----------------------|
| 🐿️ Природа | "Здесь регулярно вижу белок утром 7-9, берут орехи из рук" | Фотографы, любители природы |
| 📚 Культура | "Библиотека с редкими книгами по математике" | Студенты, книголюбители |
| 🏖️ Место отдыха | "Идеально утром 6-9: никого, чистая вода, красивый рассвет" | Ранние пташки, пловцы |
| 📸 Фото | "Лучшая точка для съёмки заката, нужен объектив 50мм+" | Фотографы |
| 🏃 Спорт | "Турники в тени, удобно заниматься утром" | Спортсмены |

#### Категории заметок

```
🐿️ Природа      — животные, растения, красивые места
📚 Культура     — библиотеки, музеи, книжные
🏃 Спорт        — турники, беговые маршруты, йога
☕ Еда          — кофейни, необычные места
📸 Фото         — лучшие точки съёмки
🎨 Творчество   — стрит-арт, мастерские
🎮 Игры        — покемоны, геокешинг
💡 Совет        — лайфхаки района
❓ Другое       — всё остальное
```

#### Механика социальной связи

```
┌─────────────────────────────────────────────────────────────────┐
│                     User Flow социальной связи                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. Пользователь находит что-то интересное                      │
│     ↓                                                           │
│  2. Создаёт ЗАМЕТКУ на карте с фото/описанием                   │
│     ↓                                                           │
│  3. Другие пользователи видят заметку                           │
│     ↓                                                           │
│  4. Ставят "Интересно" 👍 или сохраняют                         │
│     ↓                                                           │
│  5. Автор получает уведомление: "Кому-то интересно!"            │
│     ↓                                                           │
│  6. Автор решает: показать контакт или нет                      │
│     ↓                                                           │
│  7. Если да — открывается профиль/мессенджер                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### Модель данных InterestNote

### 2.3. Смешарики-напоминалки (ReminderCharacter) 🆕

**Концепция:** Игровые персонажи (в стиле Смешариков), которые напоминают о чём-то, когда пользователь подходит к заданному месту.

**Примеры:**

| Персонаж | Радиус | Напоминание |
|----------|--------|-------------|
| 🐻 Копатыч | 50м | "Купить флюс ЛТИ-120 в аптеке" |
| 🐰 Крош | 100м | "Здесь живёт друг, можно зайти" |
| 🦔 Ёжик | 30м | "Опустить письмо в почтовый ящик" |
| 🐷 Нюша | 200м | "Заказать продукты на завтра" |
| 🦉 Кар-Карыч | 150м | "Позвонить маме" |

#### Модель данных ReminderCharacter

```dart
class ReminderCharacter extends MapObject {
  final ReminderCharacterType characterType;  // Тип персонажа
  final String reminderText;                   // Текст напоминания
  final double triggerRadius;                  // Радиус срабатывания (м)
  final bool isActive;                         // Активно ли напоминание
  final DateTime? snoozedUntil;                // Отложено до
  final int triggeredCount;                    // Сколько раз сработало
}
```

#### Механика

```
1. Пользователь ставит метку с персонажем
   └── Указывает текст напоминания
   └── Указывает радиус срабатывания (10-500м)

2. Во время прогулки подходит к месту
   └── Система проверяет расстояние
   └── Если в радиусе — уведомление

3. Уведомление с персонажем
   └── "🐻 Копатыч напоминает: Купить флюс ЛТИ-120!"
   └── [Сделано] [Отложить] [Удалить]

4. После выполнения
   └── Можно удалить или оставить на будущее
```

## 3. Социальные функции

### 3.1. Профиль для контакта

Пользователь может создать профиль, который будет показан тем, кому интересны его заметки:

```dart
class ContactProfile {
  final String userId;
  final String about;                  // О себе
  final String? vkLink;                // Ссылка VK
  final String? telegramLink;          // Telegram
  final String? maxLink;               // Max (разрешённые в РФ)
  final ContactVisibility visibility;  // Кто видит контакт
  final bool acceptP2PMessages;        // Принимать P2P сообщения
}

enum ContactVisibility {
  afterApproval,     // После одобрения автора
  afterInterest,     // Все, кто поставил "Интересно"
  nobody,           // Никто (только P2P мессенджер)
}
```

### 3.2. Система "Интересно"

```dart
class NoteInterest {
  final String noteId;
  final String userId;
  final DateTime timestamp;
  final bool contactRequestSent;  // Запрос на контакт отправлен

}
```

### 3.3. P2P Мессенджер

Простейшая реализация через существующую P2P инфраструктуру:

```dart
class P2PMessage {
  final String id;
  final String fromUserId;
  final String toUserId;
  final String content;
  final DateTime timestamp;
  final bool delivered;
  final bool read;
}
```

**Поддерживаемые способы связи:**
1. **Внешние мессенджеры** — если пользователь указал VK/Telegram/Max
2. **Встроенный P2P мессенджер** — если нет внешних ссылок или пользователь разрешил

```
┌─────────────────────────────────────────────────────────────────┐
│                      Система модерации фото                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Добавление фото:                                               │

│  └── hidden    — скрыто из-за жалоб                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 5. P2P Синхронизация

### 5.1. Архитектура

```
┌─────────────────────────────────────────────────────────────┐
│                     P2P Network Topology                    │
│                                                             │
│                      Signaling Server                       │
│                    (только знакомит устройства)             │
│                            ▲                                │
│                            │ TCP (TLS + HMAC)               │
│                            │                                │
│         ┌──────────────────┼──────────────────┐            │
│         │                  │                  │            │
│         ▼                  ▼                  ▼            │
│    ┌─────────┐        ┌─────────┐        ┌─────────┐      │
│    │ Device A│◄──────►│ Device B│◄──────►│ Device C│      │
│    │  Zone 1 │  TCP   │  Zone 1 │  TCP   │  Zone 1 │      │
│    └─────────┘        └─────────┘        └─────────┘      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2. Протокол синхронизации

```
Типы сообщений:
├── sync_request     — запрос синхронизации объектов
├── sync_data        — данные объектов
├── message          — P2P сообщение
├── interest         — уведомление о "Интересно"
├── contact_request  — запрос на контакт
└── contact_approve  — одобрение контакта
```

```
TrashMonster:      ~300-500 байт

InterestNote:      ~500-1500 байт (без фото)
Photo (WebP):      ~50-100 KB (сжатое)
ReminderCharacter: ~200-400 байт
P2PMessage:        ~200-500 байт

При 1000 объектов + 100 фото: ~15-25 MB
```

## 11. Заключение

Система объектов карты превращает **Прогулкин** из простого трекера прогулок в **платформу для исследования мира и поиска единомышленников**.

Ключевые преимущества:

1. **Приватность** — нет центрального хранилища данных
2. **Интересы** — связи через общие интересы, а не просто географию
3. **Безопасность** — фото с модерацией, текст фильтруется
4. **Расширяемость** — легко добавлять новые типы объектов
5. **Офлайн** — работает без интернета
6. **Игровизация** — очки, уровни, достижения
