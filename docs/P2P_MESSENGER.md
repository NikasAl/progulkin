# P2P Мессенджер Прогулкин

## Обзор

Мессенджер в приложении «Прогулкин» работает по принципу peer-to-peer (P2P) — сообщения передаются напрямую между устройствами пользователей без использования центрального сервера.

```
┌─────────────────────────────────────────────────────────────┐
│                    Традиционные мессенджеры                 │
│                                                             │
│   Пользователь A ──→ Сервер компании ──→ Пользователь B     │
│                           ▲                                 │
│                           │                                 │
│                 Все сообщения здесь                         │
│                 (компания видит всё)                        │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                   Прогулкин (P2P мессенджер)                │
│                                                             │
│   Пользователь A ←──── P2P ────→ Пользователь B             │
│         ↑                          ↑                        │
│         │                          │                        │
│         └──────── Signaling ───────┘                        │
│                (только знакомит)                            │
│                                                             │
│   Сообщения хранятся только на устройствах пользователей    │
└─────────────────────────────────────────────────────────────┘
```

### Модели данных

#### P2PMessage

```dart
class P2PMessage {
  final String id;              // UUID сообщения
  final String fromUserId;      // ID отправителя
  final String toUserId;        // ID получателя
  final String content;         // Текст сообщения
  final DateTime timestamp;     // Время отправки
  final bool delivered;         // Доставлено ли
  final bool read;              // Прочитано ли
}
```

#### Chat

```dart
class Chat {
  final String chatId;          // ID чата (userId1_userId2)
  final String otherUserId;     // ID собеседника
  final String? otherUserName;  // Имя собеседника
  final String? lastMessageContent;  // Последнее сообщение

}
```

### Структура базы данных

#### Таблица messages

```sql
CREATE TABLE messages (
  id TEXT PRIMARY KEY,
  from_user_id TEXT NOT NULL,
  to_user_id TEXT NOT NULL,
  content TEXT NOT NULL,
  timestamp TEXT NOT NULL,
  delivered INTEGER DEFAULT 0,
  read INTEGER DEFAULT 0
);

-- Индексы для быстрого поиска
CREATE INDEX idx_msg_from ON messages(from_user_id);
CREATE INDEX idx_msg_to ON messages(to_user_id);
CREATE INDEX idx_msg_chat ON messages(from_user_id, to_user_id);
```

#### Таблица contact_profiles

```sql
CREATE TABLE contact_profiles (
  user_id TEXT PRIMARY KEY,

  visibility TEXT DEFAULT 'after_approval',
  accept_p2p_messages INTEGER DEFAULT 1
);
```

## Как начать диалог

### Способ 1: Через заметку о месте

1. Пользователь находит интересную заметку на карте
2. Нажимает кнопку **«Интересно»** на заметке
3. Владелец заметки получает уведомление
4. При одобрении — открывается возможность писать сообщения

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   Заметка "Кафе на углу"                                    │
│   ┌─────────────────────────────────────────────────────┐   │
│   │  📸 Фото места                                       │   │
│   │                                                      │   │
│   │  "Отличный кофе и уютная атмосфера"                  │   │
│   │                                                      │   │
│   │  👤 Автор: Прогульщик                                │   │
│   │                                                      │   │
│   │  [💜 Интересно]  [✅ Подтвердить]  [❌ Пожаловаться] │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
│   При нажатии "Интересно":                                  │
│   → Запрос отправляется автору                              │
│   → При одобрении — открывается чат                         │
│                                                             │

```

### Способ 2: При встрече в реальности

1. Два пользователя находятся в радиусе P2P-связи (~100м)
2. Устройства автоматически обнаруживают друг друга
3. Можно начать диалог напрямую

## Передача сообщений

### Процесс отправки

```
1. Пользователь A пишет сообщение
   └── Сообщение сохраняется в локальную БД отправителя
   └── P2PService пытается доставить напрямую

2. Если P2P соединение активно
   └── Сообщение передаётся напрямую устройству B
   └── Устройство B сохраняет в свою БД
   └── Отправляется подтверждение доставки

3. Если P2P соединения нет
   └── Сообщение остаётся в очереди
   └── Будет доставлено при следующем соединении
```

### Статусы сообщения

| Статус | Описание |
|--------|----------|
| 📤 **Отправлено** | Сообщение сохранено локально |
| ✅ **Доставлено** | Получатель получил сообщение |
| 👁️ **Прочитано** | Получатель открыл чат |

## Приватность

### Что видит сервер

**Ничего.** Signaling сервер только помогает устройствам найти друг друга:

- IP:порт устройства (временно)
- Географическая зона (geohash)
- Временный ID устройства

**Сервер НЕ видит:**
- Содержимое сообщений
- Имена пользователей
- Историю переписки
- Время сообщений

### Что хранится локально

| Данные | Где хранятся |
|--------|--------------|
| Текст сообщений | SQLite на устройстве |
| История чатов | SQLite на устройстве |
| Профиль контакта | SQLite на устройстве |
| Настройки приватности | SharedPreferences |

### Настройки приватности

Пользователь может:

- ✅ Отключить приём P2P-сообщений
- ✅ Скрыть профиль от всех
- ✅ Показывать профиль только после одобрения
- ✅ Заблокировать конкретного пользователя

## Интеграция с P2P сервисом

### Приём сообщений

```dart
// В P2PService при получении сообщения
void _onMessageReceived(Map<String, dynamic> data) {
  if (data['type'] == 'chat_message') {
    // Передаём в ChatProvider
    chatProvider.receiveMessage(data, currentUserId);
    
    // Показываем уведомление
    showNotification(
      title: data['fromUserName'] ?? 'Пользователь',
      body: data['content'],
    );
  }
}
```

### Отправка сообщений

```dart
// В ChatProvider
Future<P2PMessage> sendMessage({
  required String fromUserId,
  required String toUserId,
  required String content,
}) async {
  final message = P2PMessage.create(...);
  
  // Сохраняем локально
  await storage.saveMessage(message.toJson());
  
  // Пытаемся отправить через P2P
  p2pService.sendMessage(toUserId, message.toJson());
  
  return message;
}
```

## Ограничения

### Текущие ограничения

- ❌ Нет поддержки медиа-файлов (только текст)
- ❌ Нет редактирования сообщений
- ❌ Нет удаления сообщений у собеседника
- ❌ Нет групповых чатов

### Причины ограничений

1. **Только текст** — P2P передача файлов требует больше ресурсов
2. **Без редактирования** — сложно синхронизировать изменения P2P
3. **Без групповых чатов** — требует более сложной маршрутизации

## Безопасность

### Защита от спама

1. **Ограничение на отправку** — не более 30 сообщений/минута
2. **Требование контакта** — писать можно только после «Интересно»
3. **Блокировка пользователей** — каждый может заблокировать контакт

### Защита данных

1. **Локальное хранение** — сообщения только на устройствах
2. **Без облачного бэкапа** — данные не попадают в облако
3. **Удаление вместе с приложением** — при удалении все данные стираются

## Диагностика проблем

### Сообщения не доставляются

```
1. Проверьте P2P соединение
   └── Настройки → P2P → Статус соединения

2. Проверьте интернет
   └── P2P через интернет требует стабильной связи

3. Проверьте настройки приватности собеседника
   └── Возможно, он отключил приём сообщений
```

### История чатов пропала

```
Причины:
1. Приложение было переустановлено
   └── Данные хранятся локально и удаляются с приложением

2. Данные были очищены
   └── Настройки → Хранилище → Очистить всё

Решение:
- Используйте экспорт данных для резервного копирования
```
